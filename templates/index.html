MiniRedis

A dynamic simulation of Redis in-memory storage compared against disk-based algorithms — with real-time visualization, performance analytics, and algorithm walkthroughs.

STORAGE

In-Memory Hashing

Hash table with 16 buckets provides O(1) average-case access for SET, GET, and DEL operations.

EVICTION

LRU Cache

Least Recently Used eviction ensures the cache stays within capacity, dropping the oldest accessed key.

EXPIRY

TTL Support

Keys can expire automatically using Time-To-Live. Use SET key val EX n to set expiry.

MEMORY

Huffman Compression

Values are analyzed using Huffman encoding to estimate memory savings from compression.

DISK

Disk Simulation

A file-like sequential store using your chosen algorithm — Linear, Binary, Merge Sort, or Quick Sort.

ANALYTICS

Real-Time Graphs

All charts are generated from live execution data. No dummy values — every graph reflects actual timing.

ALGORITHMS USED
Hashing
O(1)
Core of Mini Redis. Keys mapped to buckets using a polynomial hash function.
Linear Search
O(n)
Disk baseline. Sequentially scans each record from start to end.
Binary Search
O(log n)
Requires sorted keys. Halves the search space at every step.
LRU Cache
O(1)
OrderedDict maintains insertion order; popitem evicts the front.
Merge Sort
O(n log n)
Stable divide-and-conquer sort. Guarantees worst-case performance.
Quick Sort
O(n log n)
Pivot-based in-place sort. Excellent average case, O(n²) worst.
Huffman Coding
O(n log n)
Builds a prefix-free code tree from character frequencies.
TTL Expiry
O(1)
Timestamp-based key expiration checked at access time.
